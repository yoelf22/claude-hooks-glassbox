# glassBox

A Claude Code hook that explains what every tool does while Claude works — in plain English.

When Claude uses a tool (Bash, Read, Edit, Grep, etc.), glassBox prints a brief description followed by a plain-English explanation of what the tool does and what its arguments mean. Explanations are generated by Claude and cached locally.

## Install

```bash
git clone <repo-url> && cd glassbox
./install.sh
```

Requires: `jq`, `claude` CLI, `shasum`

## How it works

glassBox registers a `PreToolUse` hook that fires before every tool call. It:

1. Reads the tool name and input
2. Normalizes the input (strips file paths, URLs, secrets — only structure remains)
3. Checks a local cache (`~/.glassbox/cache/`)
4. On miss, calls Claude to generate an explanation
5. Prints the explanation to stderr

Your data stays private — no file paths, URLs, branch names, or secrets are sent to the LLM.

## Learn / dismiss tools

Once you understand what a tool does, dismiss it:

```bash
glassbox learn Read        # dismiss the Read tool
glassbox learn Bash:git    # dismiss all git commands
glassbox learned           # list dismissed tools
glassbox unlearn Read      # re-enable full explanations
```

Learned tools still show the brief one-liner, just not the extended explanation.

## Uninstall

```bash
./uninstall.sh
```

## Cache

Explanations are cached in `~/.glassbox/cache/` and auto-expire after 30 days. Clear manually with:

```bash
glassbox clear-cache
```
